CVPR 2026 Children Gait Visual Analysis Competition in CV4CHL Workshop" - Mô tả bài toán Overview
The first AI Children Challenge aims to advance real-world applications of computer vision and AI in child healthcare, child education, and pediatrics. The 2026 CV4CHL edition featured the first track in this domain: Children Gait Visual Analysis. The main goal of Children Gait Visual Analysis is the fine-grained analysis of children's gait behaviors from keypoint sequences.

The Challenge has two tracks. In Track 1, you’ll use the provided Track 1 training set under our proposed Children Gait Pose Sequence (CGPS) Dataset to develop models that are capable of conducting fine-grained kinematic analysis to accurately predict EVGS metrics directly. In Track 2, you'll use the provided Track 2 training set under the same dataset to develop models to correctly classify the gait patterns of Bilateral Spastic Cerebral Palsy. The predicted results will be uploaded for evaluation, and the scores of the two tracks each account for 50% to determine the final score.

Description
The Challenge has two tracks. In Track 1, you’ll use the provided Track 1 training set under our proposed Children Gait Pose Sequence (CGPS) Dataset to develop models that are capable of conducting fine-grained kinematic analysis to accurately predict EVGS metrics directly. In Track 2, you'll use the provided Track 2 training set under the same dataset to develop models to correctly classify the gait patterns of Bilateral Spastic Cerebral Palsy. The predicted results will be uploaded for evaluation, and the scores of the two tracks each account for 50% to determine the final score.

EVGS Introduction
The Edinburgh Visual Gait Score (EVGS) is a clinical tool developed to visually assess gait deviations in ambulatory children using coronal and sagittal video recordings. It evaluates 17 observational scoring items for each limb that are graded on a three-point ordinal scale (0: normal, 1: moderate deviation, 2: severe deviation). Detailed scoring criteria can be found in this guide.

To simplify the problem, address the imbalance in labels, and establish a robust computational benchmark, we performed binary processing on the evaluation results, merging scores 1 and 2 into a single category.

Gait Patterns in Bilateral Spastic Cerebral Palsy


As shown in the figure, there are 4 common gait patterns in Bilateral Spastic Cerebral Palsy: True Equinus, Jump Gait, Apparent Equinus, and Crouch Gait. The other gait patterns that are relatively normal are referred to as WNL. Detailed information on the gait patterns can be found in this website.

Track 1 - EVGS Scoring
The Edinburgh Visual Gait Score (EVGS) is a kind of observational scale that is critical for diagnosing neurodevelopmental disorders such as Cerebral Palsy (CP). Therefore, designing dedicated pediatric modeling techniques and developing a model capable of automatic scoring to provide pediatricians with a reference is essential for clinical diagnosis. In this track, participants are tasked with developing algorithms capable of conducting fine-grained kinematic analysis to accurately predict EVGS metrics directly from provided children's 2D keypoint sequences.

Specifically, the participants will be provided with the training set for track 1 of our proposed CGPS dataset. For each patient, we provide EVGS scores for each limb(left/right) of the patient and a set of multi-view annotations, including keypoint sequence, bounding box, and the meta information of the original video per frame. The goal is to use the dataset to train a model to predict the EVGS scores for each limb of the patient from the evaluation set. Your final score will be based on the accuracy of each scoring item and the RMSE for the total score of the patient.

Track 2 - Classification of Gait Patterns in Bilateral Spastic Cerebral Palsy
Although Gait classification systems for children with bilateral lower limb spasticity have been developed, there are limitations to their clinical relevance and applicability in orthotic prescription. Studies have shown low levels of validity and reliability for these classification systems, emphasizing the need for comprehensive approaches that address the range and magnitude of gait deviations in children with bilateral CP. As a track running parallel to Track 1, this track focuses on dealing with this issue, which challenges teams to classify specific subtypes of bilateral cerebral palsy by recognizing subtle, pathological variations in children's gait behaviors.

Specifically, the participants will be provided with the training set for track 2 of our proposed CGPS dataset. We ensure that all the patients in this track are suffering from Bilateral Spastic Cerebral Palsy. For each patient, we provide the gait patterns for each limb(left/right) and a set of multi-view annotations with the same contents as Track 1. The goal is to use the dataset to train a model to predict the gait pattern for each limb of the patient from the evaluation set. Your final score will be based on the accuracy of each limb and the Macro F1 Score of the overall prediction.

Introduction
We introduce the Children Gait Pose Sequence (CGPS) dataset (a subset of Children Gait Video (CGV) Dataset) to support this Challenge. The dataset contains 2D keypoints annotation per frame across a total number of 339,236 frames (1,185 videos) of size 1920x1080 (2K) with 17 EVGS sub-item annotations per limb and the patient-level diagnosis results of CP motor types (Unilateral or Bilateral) and corresponding subtypes. The videos were recorded at Shenzhen Children's Hospital using smartphone cameras and action cameras, positioned simultaneously to capture sagittal and coronal views. Raw video is not provided due to the privacy protection of the child patient. In total, 110 patients participated in the data collection. In some cases, multiple recordings per view were available. The dataset provides detailed annotations per frame and per video, including the 2D human keypoints (detected with Sapiens-2B and manually adjusted by human annotators), bounding boxes (detected with SAM 3 and manually adjusted by human annotators), basic infomation of the original video, the EVGS sub-items and CP motor types and subtypes (annotated by Dr. Meihuan Huang in the organization team and reviewed by Dr. Jianguo Cao, a pediatrician with 40 years of clinical experience), the diagnosis results tracing from the patient's follow-up record.

To capture the dataset, a primary camera is positioned at the terminus of an 8-meter walkway to record the coronal (frontal and posterior) view. A secondary camera is oriented orthogonally, facing the center of the walkway, to capture the sagittal (lateral) view. This lateral camera is positioned at a sufficient distance to ensure its field of view encompasses the middle four meters of the trial space. This specific distance is calibrated to guarantee the capture of 2-3 complete gait cycles (strides) per subject.

```
🎯 Mục tiêu chính:
├── Track 1: Dự đoán EVGS Score (0/1) cho 17 items × 2 chi = 34 binary labels
├── Track 2: Phân loại 5 kiểu dáng đi (type1-4 + WNL) cho từng chi
└── Input: Sequence keypoints 2D (133 joints theo COCO-WholeBody)
```

**Dữ liệu đầu vào:**

- 110 bệnh nhân, mỗi người có 4 góc quay (trái, phải, trước, sau)
- Keypoints: 133 cặp tọa độ (x,y) + confidence score
- Meta bbox, fps, video info

### 🔹 Level 2: Hiểu sâu về domain knowledge

**Edinburgh Visual Gait Score (EVGS):**
- Công cụ lâm sàng đánh giá 17 items quan sát dáng đi 
- Mỗi item: 0 (bình thường), 1 (lệch vừa), 2 (lệch nặng) → **Binary: 0 vs {1,2}**
- Đánh giá trên 2 mặt phẳng: sagittal (nghiêng) và coronal (trước-sau)

**5 kiểu dáng đi trong Bilateral Spastic Cerebral Palsy** :
```
1. True Equinus: Đi bằng mũi chân, gối duỗi
2. Jump Gait: Gối + hông + cổ chân đều gập
3. Apparent Equinus: Trông như đi mũi chân nhưng do gối gập quá mức
4. Crouch Gait: Gối gập quá mức trong tất cả pha dáng đi
5. WNL (Within Normal Limits): Dáng đi bình thường
```

### 🔹 Level 3: Thách thức kỹ thuật nâng cao

| Thách thức | Giải pháp tiềm năng |
|-----------|-------------------|
| **Sequence dài, biến động** | Temporal modeling (Transformer, TCN, LSTM) |
| **Multi-view fusion** | Early/late fusion, attention across views |
| **Class imbalance** | Focal loss, oversampling, ensemble |
| **Small dataset (110 patients)** | Transfer learning, augmentation, cross-validation |
| **Fine-grained kinematic features** | Geometric features + learned features |
| **Evaluation metric phức tạp** | Optimize trực tiếp cho Acc + NRMSE / F1-macro |

```python
# Ví dụ: Multi-view Attention Fusion
class MultiViewFusion(nn.Module):
    def forward(self, view_features):  # [4, B, T, D]
        # Compute attention weights across views
        attn_weights = self.attention(view_features)  # [4, B, 1, 1]
        # Weighted sum
        fused = torch.sum(attn_weights * view_features, dim=0)
        return fused

# Ví dụ: Geometric Feature Engineering
def compute_kinematic_features(keypoints_seq):
    features = []
    for t in range(len(keypoints_seq)-1):
        # Joint angles (hip-knee-ankle)
        angles = compute_angles(keypoints_seq[t])
        # Angular velocity
        velocities = keypoints_seq[t+1] - keypoints_seq[t]
        # Symmetry metrics (left vs right)
        symmetry = compute_symmetry(keypoints_seq[t])
        features.append(torch.cat([angles, velocities, symmetry]))
    return torch.stack(features)
```

**Kỹ thuật nâng cao từ Kaggle Grandmasters** :
1. **Pseudo-labeling**: Dùng model mạnh generate label cho unlabeled data
2. **Test-time augmentation (TTA)**: Flip sequence, average predictions
3. **Model ensembling**: Weighted average của 3-5 models khác architecture
4. **Feature blending**: Kết hợp hand-crafted + learned features
5. **Cross-validation strategy**: Patient-level split (không leak patient giữa train/val)

```python
# Ensemble Strategy
def ensemble_predictions(models_list, data_loader, weights=None):
    all_preds_track1 = []
    all_preds_track2 = []
    
    for model in models_list:
        model.eval()
        preds1, preds2 = [], []
        with torch.no_grad():
            for batch in data_loader:
                p1, p2 = model(batch['sequence'])
                preds1.append(torch.sigmoid(p1).cpu())
                preds2.append(torch.softmax(p2, dim=1).cpu())
        all_preds_track1.append(torch.cat(preds1))
        all_preds_track2.append(torch.cat(preds2))
    
    # Weighted average
    if weights is None:
        weights = [1.0/len(models_list)] * len(models_list)
    
    final_p1 = sum(w*p for w,p in zip(weights, all_preds_track1))
    final_p2 = sum(w*p for w,p in zip(weights, all_preds_track2))
    return final_p1, final_p2
```

## 🏆 CHIẾN LƯỢC TỪ KAGGLE GRANDMASTERS

### 🔥 5 Bí quyết "winning" từ top competitors :

1. **Understand the metric deeply**
   ```python
   # Đừng optimize accuracy đơn thuần!
   # Track 1: S1 = (Acc + 1 - NRMSE)/2 → Cần cân bằng cả 2
   # Track 2: Macro F1 → Tránh bias class majority
   
   def custom_loss_track1(pred_items, pred_total, target_items, target_total):
       bce = F.binary_cross_entropy_with_logits(pred_items, target_items)
       mse = F.mse_loss(pred_total, target_total)
       return 0.6 * bce + 0.4 * mse  # Tune weight này!
   ```

2. **Patient-level validation is NON-NEGOTIABLE**
   ```python
   # SAI: Random split frames → data leakage
   # ĐÚNG: Split by patient_id
   from sklearn.model_selection import GroupKFold
   
   gkf = GroupKFold(n_splits=5)
   for train_idx, val_idx in gkf.split(X, y, groups=patient_ids):
       # Train/val không overlap patient
   ```

3. **Feature engineering > Model complexity** (với dataset nhỏ)
   ```python
   # Hand-crafted kinematic features thường giúp nhiều hơn tăng model size:
   features = {
       'joint_angles': compute_angles(keypoints),      # Hip/knee/ankle angles
       'symmetry_index': left_right_symmetry(keypoints), # L vs R difference
       'temporal_derivatives': velocity_acceleration(seq), # First/second order
       'gait_cycle_phase': estimate_phase(keypoints)   # Stance vs swing
   }
   ```

4. **Ensemble diversity > Ensemble size**
   ```
   Tốt hơn: 3 models khác nhau (LSTM + TCN + Transformer)
   Hơn là: 10 models cùng architecture với seed khác nhau
   ```

5. **Post-processing matters**
   ```python
   # Track 1: Threshold tuning per item (không dùng 0.5 cố định)
   best_thresholds = []
   for item_idx in range(34):
       best_t, best_f1 = 0.5, 0
       for t in np.arange(0.3, 0.7, 0.05):
           preds = (probabilities[:, item_idx] > t).astype(int)
           f1 = f1_score(y_true[:, item_idx], preds)
           if f1 > best_f1:
               best_f1, best_t = f1, t
       best_thresholds.append(best_t)
   ```
